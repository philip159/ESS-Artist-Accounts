# East Side Studio London - Art Submission & Product Management Platform

## Overview
East Side Studio London is a full-stack web application that streamlines the art submission process and automates e-commerce product mockup generation for platforms like Shopify. It handles high-resolution artwork uploads, detailed image analysis, print size calculations, and exports Shopify-compatible product data. The platform aims to simplify the workflow from artist submission to product creation, benefiting artists and administrators by enhancing efficiency and market reach.

## User Preferences
Preferred communication style: Simple, everyday language.

## System Architecture

### Frontend
The frontend is built with React 18, TypeScript, Vite, Wouter for routing, and TanStack Query for state management. It utilizes shadcn/ui components with Tailwind CSS for a minimalist, Material Design-inspired aesthetic. Key features include a multi-step artist submission form with server-side image analysis and an admin dashboard for managing artworks, mockups, templates, and batch exports.

### Backend
The backend uses Express.js with TypeScript, featuring session-based authentication for admin routes. Image processing, including manipulation, metadata extraction, and print size determination, is handled by Sharp. Multer manages file uploads using disk storage (`/tmp/uploads`) for artwork submissions (up to 300MB) to avoid holding large files in memory, and memory storage for smaller admin uploads. Object Storage uploads for large artwork files use streaming from disk via `uploadFileFromPath()`. A RESTful API returns JSON responses. Email notifications are managed via Resend, consolidating batch emails.

### Data Storage
The project uses PostgreSQL via Neon Database with Drizzle ORM for type-safe interactions. The schema includes tables for Artworks, Templates, Mockups, and Export Batches.

### UI/UX Decisions
The application combines Material Design principles with a Linear-inspired minimalist aesthetic, custom typography, and shadcn/ui components with Tailwind CSS for a consistent and modern user experience.

### Core Features
- **Artist & Admin Dashboards:** Manages artworks, sales, payments, onboarding invitations, and artist accounts.
- **Commission Management:** Supports global and per-artist configurable commission rates.
- **PayPal Bulk Payouts:** Facilitates creating, reviewing, approving, and processing bulk PayPal payouts to artists.
- **Email Automations:** Configurable email triggers for form submission status changes, managed by a background scheduler.
- **AR Frame Generator:** Generates 3D GLB models of framed artwork for augmented reality visualization, supporting various frame styles and mount options, integrating with Shopify product options.
- **Adaptive Product Image Shadow:** Dynamically generates product image shadows with precise geometry and scaling, consistent across live previews and server-side generation.
- **Multi-Ratio AR Images:** Admin tool to manage and assign AR images for products across multiple aspect ratios, with Dropbox integration for searching and pushing images to Shopify metafields.
- **Media Alt Text Editor:** Admin tool for bulk editing Shopify product image alt text.
- **Artwork Mount Toggle:** Allows admins to add and preview mounts for artworks, syncing `hasMount` status to Shopify.
- **Artwork Tagging:** Artists can tag artworks across categories, with AI fallback tags generated by OpenAI vision analysis if categories are left empty.
- **Product Mockup Generation:** Automated system generates product images (Black, White, Natural Frame, Unframed) at 1500x2000px, stored in Object Storage, and processed via a serialized queue. Supports artist-specific templates (`artistVendorName` on templates table) — templates with an assigned artist only generate mockups when that artist submits work; global templates (null) apply to all artists.
- **Artwork Scan Video Generation:** Automated system generates Ken Burns-style scan videos of artworks using FFmpeg, highlighting details and supporting multiple styles. Videos are stored as mockups and uploaded to Shopify as product media.
- **Scan Video Manager:** Admin tool for retroactively adding scan videos to existing Shopify products, with Dropbox search integration.
- **Product Media:** Unified admin page (`/admin/product-media`) combining media gallery, alt text editing, scan video generation, and frame mockup generation with Shopify push and position control. Replaces separate Scan Videos and Media Editor sidebar items. Supports filtering products by type, tags, and search. Mockup generation uses the existing `generateProductMockup` engine. Features batch regeneration: multi-select products via checkboxes, then batch replace frame mockups on Shopify (auto-deletes old frame-tagged and legacy-pattern mockups, generates 4 new frames from local artwork source, uploads and reorders). Single-product "Replace Mockups on Shopify" button also available in the Generate Mockups tab.
- **Scale Calibration Tool:** Admin tool on the Product Media page to calibrate auto-generated mockup scale against legacy reference images. Uses a scale multiplier (0.5x–2.0x) persisted in render settings, with real-time preview comparison and overlay blending. Accessible globally via "Scale Settings" button or per-product via the "Scale Calibration" tab. Includes a "Re-scale Existing Mockups" workflow that scans all products for auto-generated mockups (identified by alt text pattern like "Title - Black Frame"), then batch-deletes and replaces them with regenerated versions at the current saved scale, preserving media positions.
- **High-Res Review:** Admin tool (`/admin/high-res-review`) for reviewing 4:5 ratio product high-res source files. Features Pending/Reviewed tabs with Approve/Hide workflow (persisted in localStorage), Dropbox high-res file search with detected aspect ratio display (4:5, 3:4, A-Ratio, etc.), DPI-based size eligibility assessment, and batch "Add sizes to approved" action that updates artwork available sizes and pushes new size variants (with pricing from VariantConfig) directly to Shopify products.
- **Lifestyle Mockup Generation:** Perspective-aware compositing system for creating lifestyle mockups using uploaded template scenes and defined artwork zones with homography transformation.
- **Custom Product Add-ons:** Supports a three-level hierarchy for product options with configurable display types and dynamic display conditions based on Shopify variants, sizes, frames, and metafields. Mount toggle is rendered inline (same row) with the Shopify Frame picker via DOM repositioning in the widget. Addon cards use a V5a two-row layout: main row (thumbnail + name/price + description + checkbox) and an optional spec line (pipe-separated highlights from `specs` field).

## Authentication

### Admin Authentication
Session-based password authentication via Express sessions. Admins log in at `/admin/login` using `ADMIN_PASSWORD` environment variable.

### Artist Portal Pages
The artist portal (`/artist/*`) has the following pages (all require Supabase auth):
- `/artist` — My Collection (dashboard with stats: total artworks, under review, live products, units sold, revenue)
- `/artist/upload` — Artwork Upload (drag-and-drop with DPI/resolution analysis before submission)
- `/artist/commissions` — Commissions (sales by period showing gross revenue and net commission)
- `/artist/invoices` — Invoices (payout history from PayPal batch payouts)
- `/artist/faqs` — FAQs (static FAQ accordion: Getting Started, Commissions, Collection, Exclusivity)
- `/artist/settings` — Settings (display name, PayPal payout info, account info)

Legacy routes kept for backward compatibility: `/artist/artworks`, `/artist/payouts`.

Admin impersonation mirrors all pages under `/admin/view-artist/:id/*`.

Backend artist API endpoints:
- `GET /api/artist/profile` — Artist account info
- `PATCH /api/artist/profile` — Update paypalEmail, paypalRecipientName, displayName
- `GET /api/artist/artworks` — Pending artwork submissions
- `GET /api/artist/collection` — Live Shopify products
- `GET /api/artist/sales` — Sales data by period
- `GET /api/artist/commissions` — Commission history (same as sales)
- `GET /api/artist/invoices` — PayPal payout items for this artist
- `POST /api/artist/upload` — Upload new artwork file (multipart form)

### Artist Authentication (Supabase)
Artist-facing portal uses Supabase Auth (`@supabase/supabase-js`). Artists log in at `/artist/login` with:
- Email/password sign-in
- Magic link (passwordless)
- Google OAuth (if configured in Supabase dashboard)
- Password reset via Supabase-hosted email

Required environment variables (set all in Replit Secrets):

| Variable | Used by | Description |
|---|---|---|
| `VITE_SUPABASE_URL` | Frontend (Vite) | Supabase project URL — `VITE_` prefix makes it available in browser via `import.meta.env` |
| `VITE_SUPABASE_ANON_KEY` | Frontend (Vite) | Supabase public/anon key — safe to expose in browser |
| `SUPABASE_URL` | Server (Node.js) | Same Supabase project URL — used by `supabaseAdmin` in server code |
| `SUPABASE_SERVICE_ROLE_KEY` | Server (Node.js) only | Service role key — **never expose to frontend**; used for admin operations like `inviteUserByEmail` |
| `VITE_ENABLE_GOOGLE_SSO` | Frontend (Vite) | Set to `"true"` to show Google sign-in button on the artist login page. Omit/leave blank to hide it. |

The server verifies artist JWTs in `server/middleware/artistAuth.ts` using `supabaseAdmin.auth.getUser(token)`.
Admins can invite artists via the "Portal Invite" button in the admin Artists page (`POST /api/admin/artists/invite`).

## External Dependencies

### Cloud Storage
- **Replit Object Storage:** Primary file storage, integrated via Google Cloud Storage SDK.

### Third-Party Integrations
- **Dropbox Integration:** For backup and archiving via Replit Dropbox Connector.
- **Google Sheets Integration:** For exporting artwork data via Replit Google Sheets Connector.
- **Supabase:** Used exclusively for artist portal authentication (not as a database).
- **Resend:** For transactional email services and notifications.
- **Zapier:** Integrated via webhooks for post-onboarding automations.
- **PayPal Payouts API:** For processing bulk artist payouts.
- **OpenAI:** For AI fallback tag generation and social media post caption generation.
- **Postpone:** GraphQL API (`https://api.postpone.app/gql`) for auto-drafting social media posts (Instagram, LinkedIn, Threads) when artists are onboarded or upload 5+ artworks. Uses `POSTPONE_API_TOKEN` and platform username env vars (`POSTPONE_INSTAGRAM_USERNAME`, `POSTPONE_LINKEDIN_USERNAME`, `POSTPONE_THREADS_USERNAME`). Instagram requires a `mediaUrl`; LinkedIn/Threads work text-only. All posts created as `publishingStatus: "DRAFT"`. Test route: `POST /api/admin/test-postpone`.

### Export Tools
- **csv-writer:** Generates Matrixify-compatible CSV files for Shopify product imports.
